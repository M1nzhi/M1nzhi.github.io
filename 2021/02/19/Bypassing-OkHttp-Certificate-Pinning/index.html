Bypassing OkHttp Certificate Pinning

前阵子测试了一个安卓应用，发现使用了okhttp来做SSL证书绑定。我花了半天时间，尝试了各种方法来绕过SSL证书绑定，怕日后忘记，所以做个记录，日后好查阅。

#1: Xposed 模块

  1. SSLUnpinning
  2. TrustMeAlready

TrustMeAlready之前在测试其它安卓应用的时候就没成功，这次依然没成功。尝试了下SSLUnpinning，也没成功，不知道是不是操作手法不对-_-!

#2: Frida

Frida的安装很简单

  1. pip install frida-tools
  2. 下载frida-server，拷贝到安卓模拟器/data/local/tmp，并且chmod +x设置执行权限。
  3. 将burpsuite证书cert-der.crt 拷贝到安卓模拟器/data/local/tmp
  4. 将脚本Universal Android SSL Pinning Bypass注入到应用进程中


尝试了codeshare上好几个ssl pinning脚本都失败了。。

#3: 手动修改apk，重打包

分析应用日志：adb logcat ｜ grep app包

分析发现，有一串奇怪的sha1/xxxx。查了下资料，应该是证书指纹这么个东西。参考：获取SSL证书指纹

如果证书链中的证书指纹匹配到pinned中的证书指纹，则通过认证。

解压apk包，找到pinned 证书指纹的位置:/res/values/arrays.xml

将burp的证书指纹写入到arrays.xml

然后重新打包apk

最后，终于如愿以偿抓到了https数据包
